# QoE-Forecasting-Dataset

This repository contains the scripts and topology code used to run the QoE forecasting experiments as well as the dataset used to train the LSTM model. The project mixes Mininet/Containernet topologies, Docker-based edge and cloud services, and shell wrappers that automate the different experiment scenarios. 

## Experiment Scripts

Run the scripts from the repository root.

### Docker timing scripts

These scripts benchmark image pulls and container startup time:

- `bash scripts/pull_docker_image.sh`
	- Repeatedly pulls `edge_apache` and records the time in `docker_pull_times_v2.log`.
- `bash scripts/deploy_docker_container.sh`
	- Repeatedly starts and stops a container from `edge_apache:latest` and records the time in `docker_deploy_times.log`.

### Topology and experiment runners

- `bash scripts/run_multi_edge_cloud_experiment.sh`
	- Runs the multi-edge cloud scenario with planner and monitoring processes.
	- Calls `python topology/run-multi-edge-cloud.py --users=... --arr_rate=... --seed=...`.
	- Aggregates output into directories named like `logs-multi-edge-cloud-10min-40ms-30u-dyn-k5-load-plan-reactive-*`.

- `bash scripts/run_one_edge_experiment.sh`
	- Runs the single-edge scenario.
	- Uses SSH to start the steering service on a remote host, then calls `python topology/run-one-edge.py --users=... --arr_rate=... --seed=...`.

- `bash scripts/run_edge_experiment.sh`
	- Runs the edge-cloud scenario.
	- Uses `python topology/run-edge-cloud.py --users=... --arr_rate=... --seed=...`.

- `bash scripts/run_cloud_experiment.sh`
	- Runs the cloud-only scenario.
	- Uses SSH to start the steering service and container monitor on a remote machine, then calls `python topology/run-cloud-only.py --users=... --arr_rate=... --seed=...`.

## Generated Logs And Dataset

The experiments produce several artifacts that are used later for analysis and model training:

- `logs/<scenario>-<seed>/...`
	- Generated by the topology runners.
	- These folders typically contain per-run outputs such as `node_stats.csv`, `node_stats.db`, `aggregated_result.csv`, and other scenario-specific files.

## Dataset

This project uses a tabular dataset extracted from node/container monitoring logs and QoE measurements in `dataset/`.


### Notes

- The exact schema can vary slightly depending on the scenarios collected and available monitoring metrics.
- If you regenerate the dataset, keep the scenario folder naming and structure under `logs/` consistent with what the training script expects.

## Repository Layout

- `topology/` contains the simulation and experiment orchestration code.
- `services/` contains the steering and planner services used by the adaptive experiments.
- `edge/` and `cloud/` contain the containerized server-side components.
- `player/` contains the client-side playback assets and dependencies.
